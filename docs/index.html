Why image-arranger?

Character-consistent generation is a reference-image game. The generators are great — the bookkeeping around them is a spreadsheet and a folder of PNGs. This fixes that.

🏠

Local-first, zero dependencies

One Node.js file, no build step, no accounts. Your prompts and assets never leave your machine — everything lives in a workspace folder you choose, as plain JSON and files you can read and version.

✓

Adoption, not chaos

Every output is a candidate; you adopt the good ones. Adopted images become the canonical references for the next round — per part, per character — and source links always resolve to the current canonical at queue time.

🤖

Any generator, human or agent

Requests are JSON files, not API calls. Process them yourself in ChatGPT, Midjourney or Vidu — or hand the queue to a coding agent (AGENTS.md included) and review what comes back.

The tab order is the workflow

From one key visual to a consistent character across stills and video — every step feeds the next.

01 · CREATE KIT

Identity sheet

One-shot generate the character's canonical sheet from a reusable prompt template. Fix one part without rerolling everything: decompose into per-part references.

02 · BASE

Part references

Manage per-part reference entries (face, expression, outfit…) with candidates. Only approved ones get adopted.

03 · IMAGE

One prompt, one output

Compose prompts that attach adopted images as source inputs. Improve or retry; adoption stays exclusive.

04 · VIDEO

Frames for i2v

Point start/end frames at adopted images for image-to-video services like Vidu.

05 · QUEUE

Requests as files

Each request is a JSON file with prompt, references and output dir. Edit, cancel or complete from the UI.

Quick Start

Requires Node.js 20+. No dependencies, no build step.

# clone & run
git clone https://github.com/aiarranger/image-arranger.git
cd image-arranger
node server.mjs --workspace ./workspace/demo --init sample --port 4217
# then open http://127.0.0.1:4217/

Before publishing a workspace, run --doctor: it scans for secret-like strings, local paths and missing provenance fields. Every asset records license, AI-generated and human-reviewed flags.

🇯🇵 日本語

image-arranger は「生成しない」画像・動画生成ワークフロー管理ツールです。 どのリファレンスを正とするか、どの候補を採用したか、何を依頼中か——生成サービス側が持っていない管理層をローカルで完結させます。

  • 素材作成：採用画像を参照に、キャラの正となるリファレンスシートをテンプレプロンプトで一発生成。一部だけ直したいときはパーツ分解→そのパーツだけ改善→再生成
  • ベース：パーツ別リファレンスを候補管理し、承認したものだけ「採用」に
  • 画像：1プロンプト＝1アウトプット。採用済み画像を元画像として添付（リンクはキュー時に最新の「正」へ解決）
  • 動画：採用画像を start/end フレームに指定して image-to-video へ
  • キュー：依頼はすべて JSON ファイル。人間でもコーディングエージェントでも処理できます
